상호작용성 추가하기

화면 구성요소들은 유저 입력에 반응하여 업데이트됩니다. 예를 들어 이미지 갤러리를 클릭하는 것은 동적인 이미지로 변경합니다 React에서 시간에 따라 변경되는 데이터를 state 라고 부릅니다. 여러분은 어떤 컴포넌트에도 state를 추가할 수 있으며 필요에 따라 업데이트할 수 있습니다. 이 챕터에서, 상호작용을 관리하고 상태를 업데이트하고 시간 흐름에 따라 다른 출력을 표현하는 컴포넌트를 작성하는 법을 배울 것입니다.

In this chapter
  • 유저가 만드는 이벤트를 처리하는 방법
  • state로 컴포넌트들이 정보를 기억하도록 하는 방법
  • React가 UI를 두 단계로 업데이트 하는 방법
  • state가 변경된 후 바로 업데이트되지 않는 이유
  • 여러 state 업데이트가 저장되는 방법
  • state의 객체를 업데이트하는 방법
  • state의 배열을 업데이트하는 방법

이벤트에 응답하기​

React는 여러분이 event handler 를 여러분의 JSX에 추가할 수 있게 해줍니다. Event handler들은 클릭, 마우스 올리기, 입력창 포커싱 등과 같은 유저 상호작용에 반응하는 함수들입니다.

<button>과 같은 내장 컴포넌트들은 onClick과 같은 브라우저 내장 이벤트들만 지원합니다. 하지만 여러분은 여러분만의 컴포넌트를 만들 수 있고 그들의 Event handler에 여러분이 선호하는 구체화된 기능이 담긴 이름들을 지어줄 수 있습니다.

State: 컴포넌트의 메모리​

컴포넌트들은 종종 상호작용의 결과로 화면상에 표시되는 것을 바꿀 필요가 있습니다. 입력창에 타이핑할때 입력 영역이 업데이트되어야하고, 이미지 카라셀의 "다음"을 클릭했을 때 표시되는 이미지를 변경하는 것, "구매" 버튼을 클릭해 장바구니에 물건을 추가하는 것이 있을 것 입니다. 컴포넌트는 현재 입력 값, 현재 이미지, 장바구니를 "기억"해야합니다. 이와 같은 컴포넌트에서 구체화된 메모리를 state 라고 부릅니다.

여러분은 useState 훅을 이용하여 컴포넌트에 state를 추가할 수 있습니다. Hooks 는 여러분의 컴포넌트가 React 기능들(state는 이 기능들 중 하나)을 사용할 수 있게해주는 특별한 함수들입니다. useState 훅은 state 변수들을 선언할 수 있도록 해줍니다. 초기 state를 받아 value의 쌍을 반환합니다: 현재 state와 그것을 업데이트할 수 잇도록 해주는 state 설정 함수입니다.

const [index, setIndex] = useState(0);
const [showMore, setShowMore] = useState(false);

렌더링과 커밋​

여러분의 컴포넌트가 스크린에 표현되기 전에, 컴포넌트들은 React에 의해 렌더링되어야합니다. 이 과정에서 단계를 이해하는 것은 코드가 어떻게 동작하는지 여러분이 생각하는 데 도움이 될 것이고 그 동작을 설명할 수 있게 해줍니다.여러분의 컴포넌트가 스크린에 표현되기 전에, 컴포넌트들은 React에 의해 렌더링되어야합니다. 이 과정에서 단계를 이해하는 것은 코드가 어떻게 동작하는지 여러분이 생각하는 데 도움이 될 것이고 그 동작을 설명할 수 있게 해줍니다.

여러분의 컴포넌트들이 부엌에 있는 요리사들이라고 상상하고 재료들로 맛있는 음식들을 만들고 있다고 상상해보세요. 이 시나리오에서 React는 고객들의 요청을 넣고 그들의 주문을 가져다주는 역할을 하는 웨이터입니다. UI를 요청하고 서빙하는 과정은 세가지 단계를 거칩니다.

  1. 렌더링 트리거(부엌에 주문 전달)
  2. 컴포넌트 렌더링(부엌에서 주문을 준비함)
  3. Dom에 커밋하기(테이블에 주문한 것을 올림)

스냅샷과 같은 State​

Javascript 변수들과는 다르게, React의 state는 다소 스냅샷처럼 동작합니다. 여러분이 가진 값을 세팅시에 바로 바꾸지 않는 대신, 재렌더링이 트리거됩니다. 처음엔 놀라울 것 입니다.

console.log(count); // 0
setCount(count + 1); // Request a re-render with 1
console.log(count); // Still 0!

React는 여러분이 작은 버그들을 피하도록 돕기위해 이와같이 동작합니다. 여기에 작은 채팅 앱이 있습니다. 여러분이 처음 "보내기"를 누르고 다음 수신인을 Bob으로 바꿨을때 어떤일이 일어나는지 맞춰보세요. 5초 후 누구의 이름이 alert될까요?

일련의 state 변화를 저장하는 법​

이 컴포넌트는 버그를 갖고 있습니다: "+3"을 클릭하면 점수를 한번 올립니다.

스냅샷과 같은 state는 왜 이렇게 동작하는지 설명해줍니다. state를 세팅하는 것은 실행중인 코드에서 아직 바뀌지 않은 값에 대해 새로운 재렌더링을 요청하게됩니다. 따라서 score는 여러분이 setScore(score+1)을 호출한 뒤에도 0이 됩니다.

console.log(score); // 0
setScore(score + 1); // setScore(0 + 1);
console.log(score); // 0
setScore(score + 1); // setScore(0 + 1);
console.log(score); // 0
setScore(score + 1); // setScore(0 + 1);
console.log(score); // 0

여러분은 updater function 을 state를 세팅할때 넘김으로서 값을 고정할 수 있습니다. setScore(score+1)을 setScore(s => s+1)로 수정하는 것이 "+3"버튼을 어떻게 고치는지 알아봅시다. 이 방법은 여러분이 여러 state 업데이트를 저장해야한다면 번거로울 수 있습니다.

state에서 객체 업데이트하기​

State는 객체를 포함한 어떠한 Javscript 값도 저장할 수 있습니다. 하지만 여러분은 React state가 가지고 있는 객체나 배열을 바로 변경하면 안됩니다. 대신에 객체나 배열을 업데이트 하고 싶을 때에는 새로운 것을 만들어서(혹은 복사본이나), 그 복사본으로 state를 업데이트 하여야합니다.

코드에서 객체들을 복사하는 것이 싫다면, 반복되는 코드를 줄이기 위해 Immer와 같은 라이브러리를 활용할 수 있습니다.

state에서 배열 업데이트하기​

배열은 여러분이 state에 저장할 수 있고 읽기전용으로 처리해야하는 또 다른 변경가능한 Javascript 객체 타입입니다. 객체들과 같이, state에 저장된 배열을 업데이트하기를 원한다면 새로운 것을 만들어서(혹은 복사본이나) 새 배열을 state를 세팅하는데 사용하여야 합니다.

만약 코드에서 배열을 복사하는 것이 싫다면, 반복되는 코드를 줄이기 위해 Immer와 같은 라이브러리를 활용할 수 있습니다.
